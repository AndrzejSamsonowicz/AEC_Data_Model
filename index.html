Example 2: Material Usage Analysis

Instantly see which materials are used across every file in your hub — no file-opening required. Each file is scanned with a single API call that returns material counts for thousands of elements in milliseconds.

💡 AEC Data Model Advantage: The distinctPropertyValuesInElementGroupByName query returns a full material breakdown per file in one call — no element iteration or file download required.

Material Distribution

Scanning files...
GraphQL Query 
query GetMaterialDistribution($elementGroupId: ID!, $name: String!, $filter: ElementFilterInput) {
  distinctPropertyValuesInElementGroupByName(
    elementGroupId: $elementGroupId
    name: $name
    filter: $filter
  ) {
    results {
      definition { id }
      values(limit: 500) {
        value
        count
      }
    }
  }
}

Variables:
{
  "elementGroupId": "YWVjZH5...",
  "name": "Structural Material",
  "filter": {
    "query": "property.name.category==Walls"
  }
}

✨ Key Benefit: One call per file — returns material breakdown for ALL elements in that category.
⚡ Speed: No element iteration needed — the API aggregates counts server-side.
📊 Run once per file, aggregate across your portfolio for instant cross-hub material analysis.

Example 5: Smart Folder Search

Navigate your project folder tree and instantly search for elements across every model inside — no file-opening required. Scope your query to an exact folder or drill recursively through all subfolders.

💡 AEC Data Model Advantage: elementsByFolder and elementsByFolderAndSubFolders let you scope a search to any folder path in your project — queries run server-side across all models simultaneously.
Select a project to browse folders

Elements by File

Searching folder…
GraphQL Query 
# SHALLOW mode (exact folder only):
query SearchFolder(
  $projectId: ID!
  $folderId:  ID!
  $filter:    ElementFilterInput
  $pagination: PaginationInput
) {
  elementsByFolder(
    projectId:  $projectId
    folderId:   $folderId
    filter:     $filter
    pagination: $pagination
  ) {
    pagination { pageSize cursor }
    results { id elementGroup { id name } }
  }
}

# RECURSIVE mode (folder + all subfolders):
# Step 1 — discover all files under the folder tree:
query GetEGsInFolderTree($projectId: ID!, $folderId: ID!) {
  elementGroupsByFolderAndSubFolders(
    projectId: $projectId
    folderId:  $folderId
  ) {
    results { id name }
  }
}
# Step 2 — count elements per file (run once per EG from Step 1):
query CountEls($elementGroupId: ID!, $filter: ElementFilterInput, $pagination: PaginationInput) {
  elementsByElementGroup(
    elementGroupId: $elementGroupId
    filter:         $filter
    pagination:     $pagination
  ) {
    pagination { cursor }
    results { id }
  }
}

✨ Shallow: elementsByFolder — groups returned elements by file.
📂 Recursive: elementGroupsByFolderAndSubFolders discovers all files,
   then elementsByElementGroup counts per file in parallel batches.
🔄 Cursor pagination: up to 500 elements counted per file.
